# PDF Rename Expert — інструкція для користувача

Документ описує, як підготувати комп’ютер, запустити програму й користуватися нею для перейменування PDF за даними з Excel.

---

## 1. Два способи роботи

| Спосіб | Коли підходить |
|--------|----------------|
| **Готовий `PDF_Rename_Expert.exe`** | Не потрібно встановлювати Python. Достатньо скопіювати папку з exe та необхідними файлами (див. нижче). |
| **Запуск через Python** | Якщо ви розробляєте/оновлюєте скрипт або не використовуєте зібраний exe. |

На **Windows 7** для exe або для Python рекомендується середовище з **Python 3.8.x** (див. розділ 8).

---

## 2. Готовий файл програми (`PDF_Rename_Expert.exe`)

### 2.1. Що покласти поруч із exe

У **одній папці** з `PDF_Rename_Expert.exe` мають бути:

- Файл Excel із базою споживачів (див. розділ 6).
- Підпапки для сканів (програма може створити їх сама при першому відкритті):
  - `Scan\Населення\`
  - `Scan\Промислові\`

Після роботи поруч можуть з’явитися:

- `pdf_rename_expert.log` — технічний журнал;
- `pdf_rename_suffix_history.json` — збережені варіанти поля «Належність лічильника».

### 2.2. Якщо exe не запускається

1. Встановіть **[Microsoft Visual C++ Redistributable](https://learn.microsoft.com/en-us/cpp/windows/latest-supported-vc-redist)** відповідної розрядності (**x64** або **x86** — як і ваш exe).
2. На Windows 7 переконайтеся, що встановлені оновлення системи, зокрема компоненти для підтримки сучасних програм (**Universal CRT**, наприклад оновлення **KB2999226**, якщо їх ще не ставили).

---

## 3. Встановлення Python (якщо запускаєте `pdf_rename_expert.py`)

### 3.1. Завантаження

1. Відкрийте сайт [python.org/downloads](https://www.python.org/downloads/).
2. Для **Windows 10/11** підійде актуальна стабільна версія Python 3 (наприклад 3.11–3.13).
3. Для **Windows 7** офіційно підтримується лише **[Python 3.8.10](https://www.python.org/downloads/release/python-3810/)** — завантажте інсталятор **Windows x86-64** або **Windows x86** (32-біт), залежно від вашої системи.

### 3.2. Під час інсталяції (Windows)

- Увімкніть опцію **«Add python.exe to PATH»** / **«Додати Python до PATH»**.
- Натисніть **Install Now** (або «Налаштувати» й підтвердіть PATH).

### 3.3. Перевірка в командному рядку

Відкрийте **cmd** або **PowerShell** і виконайте:

```text
python --version
```

Або, якщо на комп’ютері кілька версій Python (рекомендовано для Win7):

```text
py -3.8 --version
```

Має з’явитися рядок із номером версії (наприклад `Python 3.8.10`).

### 3.4. Оновлення інструмента `pip` (за бажанням)

```text
python -m pip install --upgrade pip
```

Для конкретної версії:

```text
py -3.8 -m pip install --upgrade pip
```

## 4. Встановлення бібліотек (залежностей)

Основні залежності для роботи програми — у файлі **`requirements.txt`**. Для самостійної збірки `.exe` потрібні також пакети з **`requirements-dev.txt`** (зокрема PyInstaller).

### 4.1. Перехід у папку проєкту

У **cmd**:

```text
cd /d C:\шлях\до\PDF_Rename
```

(Замініть шлях на реальний, де лежить `pdf_rename_expert.py`, `app_ui.py`, `naming.py`, `excel_db.py` та файли `requirements*.txt`.)

### 4.2. Встановлення пакетів однією командою

```text
python -m pip install -r requirements.txt
```

Для **Python 3.8** явно:

```text
py -3.8 -m pip install -r requirements.txt
```

Це встановить зокрема: **pandas**, **openpyxl**, **tkcalendar**, **PyMuPDF** (`pymupdf`), **Pillow**. PyInstaller до цього списку не входить.

Щоб додатково встановити залежності для самостійної збірки `.exe`:

```text
python -m pip install -r requirements-dev.txt
```

або одразу обидва файли:

```text
python -m pip install -r requirements.txt -r requirements-dev.txt
```

### 4.3. Якщо потрібно лише запуск без збірки exe

Достатньо `requirements.txt` або вручну без PyInstaller:

```text
python -m pip install pandas openpyxl tkcalendar pymupdf Pillow
```

## 5. Запуск програми з коду

Переконайтеся, що ви в папці проєкту (усі файли `.py` програми мають бути в одній папці):

```text
python pdf_rename_expert.py
```

або:

```text
py -3.8 pdf_rename_expert.py
```

Відкриється вікно програми з графічним інтерфейсом.

## 6. Excel-база споживачів

- Файл **краще називати** так: `Споживачі ЕЕЛ-2 (ЕЧ-2).xlsx` і класти **в ту саму папку**, що й exe або скрипт. Якщо такого імені немає, програма спробує взяти **перший знайдений `.xlsx`** у цій папці.
- Аркуш у книзі має називатися **`база`** (маленькими літерами, як у типовій конфігурації).

Якщо файлу немає або він зайнятий іншою програмою (наприклад відкритий у Excel), пошук за EIC може видавати попередження — закрийте файл у Excel або скопіюйте базу в папку програми.

Натисніть **«Завантажити базу»**, щоб явно перечитати Excel у пам’ять (при старті завантаження також виконується автоматично без зайвих діалогів успіху).

## 7. Короткий опис роботи в програмі

### 7.1. Підготовка файлів

1. Покладіть потрібні PDF у:
   - `Scan\Населення\` або  
   - `Scan\Промислові\`  
   залежно від обраного джерела у верхній панелі (**Населення** / **Промислові**).
2. Натисніть **«Оновити список (F5)»**, щоб побачити файли у списку ліворуч.

### 7.2. Пошук споживача

1. У полі **EIC (останні 5+ символів)** введіть щонайменше **5 символів** з кінця EIC.
2. Якщо база завантажена, з’явиться список збігів — оберіть потрібний рядок.
3. Поля **Назва**, **Адреса**, повний **EIC** підставляться з обраного запису (за потреби відредагуйте вручну).

### 7.3. Дані документа

Заповніть **дату**, за потреби позначте **Інвест**, **№ лічильника**, **фазу**, **вид роботи**, **вид перевірки**. Для робіт з **АСКОЕ** з’являться поля **IP** та **№ модему**.

У блоці **Ім’я PDF (прев’ю)** відображається майбутня назва файлу та його довжина (обмеження для Windows ураховані).

### 7.4. Перейменування

1. Оберіть PDF у списку зліва.
2. Переконайтеся, що у прев’ю немає критичних помилок (програма покаже попередження в рядку статусу).
3. Натисніть **«Обробити»** або скористайтеся **Ctrl+Enter**.

Файл перейменовується **у тій самій папці**. Якщо таке ім’я вже існує, буде додано суфікс виду `(1)`.

### 7.5. Експорт у Excel

Кнопка **«Експорт в Excel»** формує звіт із перейменованих PDF (розбір імен файлів) по папках `Scan\Населення` та `Scan\Промислові`. Файл звіту зберігається **в папці програми**; якщо файл уже є, програма запитає про перезапис.

### 7.6. Корисні дії

- **F5** — оновити список PDF.
- **Ctrl+Enter** — обробити обраний файл.
- Для деяких полів і списків працюють **Ctrl+A** (виділити все) та **Ctrl+C** (копіювати рядок у списку).

## 8. Особливості Windows 7

- Використовуйте **Python 3.8.10** для запуску з коду або для збірки exe.
- Збірка одним файлом у проєкті: скрипт **`build_win7.bat`** або команда `pyinstaller --noconfirm PDF_Rename_Expert.spec`. Перед цим виконайте `pip install -r requirements.txt -r requirements-dev.txt`.
- Обов’язково встановіть **Visual C++ Redistributable** та актуальні оновлення системи (див. розділ 2.2).

## 9. Типові проблеми

| Проблема | Що зробити |
|----------|------------|
| «Не знайдено Excel-базу» | Покладіть `.xlsx` у папку з програмою; перевірте наявність аркуша `база`. |
| «Файл Excel зайнятий» | Закрийте книгу в Microsoft Excel або іншій програмі. |
| Не відкривається прев’ю PDF | Перевірте встановлення **PyMuPDF** та **Pillow** (`pip install pymupdf Pillow`). |
| Помилки під час `pip install` | Запустіть командний рядок **від імені адміністратора** або додайте `--user`: `python -m pip install --user -r requirements.txt`. |
| Довге завантаження великої бази | Зачекайте завершення індикатора біля «Завантажити базу»; не відкривайте той самий файл у Excel паралельно. |
